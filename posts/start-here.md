# Start Here

This site reads Markdown files from the `posts/` folder and turns them into polished article pages in the browser. There is no build step, database, or server code.

## Add your next post

- Create a Markdown file in `posts/`
- Add the post metadata to `posts.json`
- Commit and push the files to GitHub
- Turn on GitHub Pages for the repository

Your post URL will look like this:

```text
https://your-name.github.io/your-repo/#/post/start-here
```

> Static hosting works best when the page route stays inside the hash.

You can edit the title, description, tags, cover image, and reading time in `posts.json`.
