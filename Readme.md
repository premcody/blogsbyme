 Right now the site uses marked, which supports a strong Markdown feature set, especially GitHub Flavored Markdown:

  - Headings: #, ##, ###
  - Paragraphs
  - Bold and italic
  - Links
  - Images
  - Blockquotes
  - Ordered and unordered lists
  - Nested lists
  - Code blocks
  - Inline code
  - Horizontal rules
  - Tables
  - Task lists: - [x]
  - Strikethrough: ~~text~~
  - Autolinks
  - Raw HTML, but sanitized by DOMPurify
